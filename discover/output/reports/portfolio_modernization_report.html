Executive Summary

▼
🔴 Critical Finding: Both applications run on IBM AIX 7.2, whose standard support ended April 30, 2023. The portfolio is running on an unsupported operating system with no security patch stream — this is an immediate compliance and security risk.
🟡 Significant Finding: COBOL-2014 (legacy language) and Oracle 19c (Extended Support through Jan 2027) affect all portfolio applications. Oracle license costs are actively incurred.
💡 Opportunity: The portfolio has an estimated $257,760 net benefit over 3 years from actionable modernization scenarios, requiring a total investment of approximately $815,640.

Key Statistics

  • Portfolio size: 2 applications (both In-Scope, Production)
  • Business domain: Finance (Financial Planning, Accounting, Budgeting)
  • Technology stack: COBOL-2014 on IBM AIX 7.2 + Oracle 19c (identical for all apps)
  • Architecture: 1-Tier monolith — no APIs, no containers, no CI/CD
  • EOL components: IBM AIX 7.2 (standard support ended April 2023)
  • Outdated components: COBOL-2014 and Oracle 19c (Extended Support)
  • Applicable modernization scenarios: 6 per application (out of 10 evaluated)

Modernization Opportunities

▼
🔧 OS Modernization
Priority: High | Both Applications
Migrate from EOL IBM AIX 7.2 to enterprise Linux (RHEL 9 or Ubuntu 22.04 LTS). This is the foundational prerequisite for cloud migration, containerization, and all other modernization paths. Estimated savings: $400/year per app in licensing and operational costs.
🗄️ Database Modernization
Priority: High | Both Applications
Oracle 19c is in Extended Support (extra fees). Two options: (1) Upgrade to Oracle 23ai for continued vendor support; (2) Migrate to PostgreSQL to eliminate the commercial license entirely. PostgreSQL migration yields ~$10,000/year savings per app in license costs alone on 1TB databases.
☁️ Cloud Migration
Priority: High (Partially Applicable) | Both Applications
Currently On-Premise. After Linux migration: standard lift-and-shift to AWS/Azure/GCP becomes available. Interim option: IBM Power Virtual Server for AIX-compatible cloud hosting. Cloud migration delivers $3,000/year savings per app in infrastructure run costs.
🏗️ Application Refactoring
Priority: High | Both Applications
1-Tier COBOL monolith decomposition into microservices. Highest ROI scenario with $150,000/year savings per app (license + run cost + operational). Requires COBOL-to-modern-language conversion as a prerequisite. Critical given 2027 decommission targets.
💻 COBOL Modernization
Priority: High | Both Applications
COBOL-2014 represents extreme talent risk. Automated COBOL-to-Java or COBOL-to-.NET conversion tools (IBM Rational, Micro Focus, TSRI) can accelerate this transition. No finance model available for this scenario — bespoke scoping required. Essential for 2027 decommission timeline.
📊 Observability Implementation
Priority: Medium | Both Applications
Neither application has logging or monitoring solutions. Implementing observability (ELK Stack, Prometheus/Grafana, or Datadog) is a risk reduction measure that should accompany OS migration. Not modeled in scenarios but should be included in any modernization project budget.

Roadmap Proposal

▼
Phase 1 — Immediate (Q3 2026)
▸ Assess IBM AIX 7.2 Extended Support contract status
▸ Evaluate Oracle 19c Extended Support costs and upgrade path
▸ Initiate Linux OS migration planning (RHEL 9 / Ubuntu 22.04)
▸ Implement CI/CD pipeline for both applications
Phase 2 — Short Term (Q4 2026 — Q2 2027)
▸ Execute Linux OS migration (COBOL runtime validation on Linux)
▸ Deploy monitoring and logging solutions (Prometheus + ELK or equivalent)
▸ Begin Oracle → PostgreSQL migration assessment and test environment setup
▸ Plan IBM Power Virtual Server or cloud migration post-Linux
Phase 3 — Medium Term (Q3 2027 — Q1 2028)
▸ Execute PostgreSQL migration (1TB Oracle data, phased approach)
▸ Execute cloud migration to AWS/Azure/GCP (post-Linux)
▸ Begin COBOL modernization (automated conversion tooling evaluation)
▸ Implement managed database services (RDS/Azure DB/Cloud SQL)
Phase 4 — Long Term (Q2 2028 — Q4 2028)
▸ Complete COBOL → Java/.NET modernization
▸ Application refactoring: decompose 1-Tier monolith into services
▸ Enable containerization (Kubernetes deployment)
▸ Full cloud-native target state achieved
