## 2. What the "java-update" Scenario Means for This Project

The `java-update` scenario is **not a language version upgrade** (the code is already at Java 21 best practices). Instead it has three distinct dimensions:

1. **Documentation currency** — `arc42-documentation.md` was clearly authored for the *original* trivial 5-line HelloWorld. Every section describes a single-class, single-method, zero-dependency program with no tests and no build tool — all factually wrong.

2. **Cookbook compliance** — Two mandatory cookbook guidelines are not yet applied:
   - `ai-generated-comment-guideline.md` — requires the exact comment `generated by AI in Github cloud` near the top of new/modified files and directly above each new class.
   - `mermaid-architecture-documentation-guideline.md` — requires system context, component interaction, and runtime flow diagrams.

3. **Duplicate file hygiene** — `HelloWorld.java` at the repository root is an unmanaged copy of `src/main/java/HelloWorld.java`.
